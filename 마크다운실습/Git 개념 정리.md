### Git 개념 정리

1. **working directory**

    파일의 변경사항

   -> add

2. **staging area** 버전으로 기록하기 위한 파일 변경사항의 목록

   : 워드나 엑셀 같은 오피서 문서 작업 중 갑자기 꺼지더라도 임시저장된 문서파일을 사용자에게 보여주는 것과 비슷한 개념? 

   -> commit

3. **repository** 커밋(버전)들이 기록되는곳 



#### 기본명령어 log

- 현재 저장소에 기록된 커밋을 조회
- 다양한 옵션을 통해 로그를 조회할 수 있음
